项目经验

2019.04-2019.06    旅客行程智能推荐系统    团队开发

技术栈：SpringBoot+SpringMVC+MyBatis+Tomcat+MySQL+Maven+Git+Dubbo+Swagger2+Nginx

开发工具：IDEA+Navicat

项目描述：该系统由前端组、后端组、算法组联合开发，主要分为用户模块、评论模块、购票人模块和路线推荐 模块，旨在为旅客提供最优的行程推荐；

负责模块：

1.用户模块：用户注册，用户登录，密码找回，修改基本信息，上传头像等功能

2.评论模块：用户可以对推荐出来的路线进行评价，使推荐算法更加完善

3.购票人模块：包括购票人的绑定、解绑、修改购票人信息、显示绑定购票人等功能

技术要点：

1.采用 Dubbo 分布式技术和 Zookeeper 注册服务以及负载均衡机制，将程序分为多个模块并部署在不同的服 务器上，提高了系统的并发量

2.采用 nginx 搭建静态资源服务器，前端通过 Ajax+formdata 将数据传输到后台，Java 通过 sftp 协议将文件 上传到服务器，实现上传头像功能。

3.配置 Druid 连接池，提升数据库的访问效率，使用 MyBatis 动态 SQL 进一步提高开发效率

4.使用短信验证来注册用户

5.调用百度地图 API 获取本地交通数据

6.使用 swagger 来提供实时接口文档

7.前后端分别通过相应配置，来解决跨域问题

项目总结：

1.软件开发一定要先写好开发文档，文档能够提高开发效率，使团队开发具有协调性。

2.团队开发需要不断的沟通与探讨，及时发现问题并提出自己的解决方案。

3.学会查找网络资源，学习新技术，自学能力的培养很重要。

4.算法很重要，它不仅是软件的核心，而且一个好的算法能够大大提高软件的效率。
